# <a name="manage-gdpr-data-subject-requests-with-the-dsr-case-tool-in-the-security--compliance-center"></a>보안 및 준수 센터에서 DSR 사례 도구를 사용하여 GDPR & 요청 관리

EU GDPR(일반 데이터 보호 규정)은 EU(유럽 연합) 내에서 개인 정보 보호 권한을 보호하고 활성화하는 것입니다. GDPR은 유럽 연합(데이터 주체)의 개인에게 개인 데이터의 액세스, 검색, 수정, 삭제 및 처리 제한 권한을 제공합니다. GDPR에 따라 개인 데이터는 식별되거나 식별 가능한 자연인과 관련된 모든 정보를 의미합니다. 개인 데이터에 대한 작업을 수행하기 위해 조직에 사람이 공식적으로 요청하는 요청을 데이터 주체 요청 또는 DSR이라고 합니다. Office 365에서 데이터에 대한 DSRS에 응답하는 데 대한 자세한 내용은 [Office 365 데이터 주체 요청 가이드를 참조하세요.](https://go.microsoft.com/fwlink/?linkid=871169 )
  
조직 내 사람이 제출한 DSR에 대한 응답으로 조사를 관리하려면 보안 및 준수 센터의 DSR 사례 도구를 사용하여 & 저장된 콘텐츠를 찾을 수 있습니다.
  
- 조직의 모든 사용자 사서함 여기에는 비즈니스용 Skype 대화 및 Microsoft Teams의 일대일 채팅이 포함됩니다.
    
- Microsoft 365 그룹과 연결된 모든 사서함 및 Microsoft Teams의 모든 팀 사서함
    
- 조직의 모든 SharePoint Online 사이트 및 비즈니스용 OneDrive 계정
    
- 조직의 모든 Teams 사이트 및 Microsoft 365 그룹 사이트
    
- Exchange Online의 모든 공용 폴더
    
DSR 사례 도구를 사용하여 다음을 할 수 있습니다.
  
- 각 DSR 조사에 대한 별도의 사례를 만들 수 있습니다.
    
- DSR 사례에 액세스할 수 있는 사용자 제어를 위해 사례의 구성원으로 추가합니다. 구성원만 사례에 액세스할 수 있으며 보안 및 준수 센터의 **DSR** 사례 페이지에 있는 사례 목록에서 & 수 있습니다. 또한 동일한 사례의 서로 다른 구성원에게 서로 다른 사용 권한을 할당할 수 있습니다. 예를 들어 일부 구성원은 사례 및 검색 결과만 볼 수 있도록 허용하고 다른 구성원이 검색을 만들고 검색 결과를 내보낼 수 있도록 허용할 수 있습니다. 
    
- 기본 제공 검색을 사용하여 특정 데이터 주체가 만들거나 업로드한 모든 콘텐츠를 검색합니다.
    
- 선택적으로 기본 제공 검색 쿼리를 변경하고 검색을 다시 실행하여 검색 결과 범위를 좁힐 수 있습니다.
    
- DSR 사례와 연결된 다른 콘텐츠 검색을 추가합니다. 여기에는 Office 로밍 서비스에서 부분적으로 인덱싱된 항목 및 시스템 생성 로그를 반환하는 검색 만들기가 포함됩니다.
    
- DSR 액세스 또는 내보내기 요청에 대한 응답으로 데이터를 내보낼 수 있습니다.
    
- DSR 조사 프로세스가 완료되면 사례를 삭제하세요. 그러면 사례와 연결된 모든 검색 및 내보내기 작업이 제거됩니다.
    
DSR 사례 도구를 사용하여 DSR 조사를 관리하는 고급 프로세스는 다음과 같습니다.
  
[Step 1: Assign eDiscovery permissions to potential case members](#step-1-assign-ediscovery-permissions-to-potential-case-members)

[3단계: 검색 쿼리 실행](#step-3-run-the-search-query)

[4단계: 데이터 내보내기](#step-4-export-the-data)

[(선택 사항) 5단계: 기본 제공 검색 쿼리 변경](#optional-step-5-revise-the-built-in-search-query)

[DSR 사례 도구 사용에 대한 자세한 정보](#more-information-about-using-the-dsr-case-tool)
  
> [!IMPORTANT]
> 이 도구는 관리자가 DSR 사례 도구에 있는 기본 제공 검색 및 내보내기 기능을 활용할 수 있도록 하여 DSR 액세스 또는 내보내기 요청을 수행하는 데 도움을 줄 수 있습니다. 이 도구를 사용하면 데이터 주체가 제출한 DSR 요청과 관련된 데이터를 내보낼 수 있는 최선의 방법을 사용할 수 있습니다. 그러나 검색 결과는 데이터 주체 또는 항목 내보내기 목적으로 항목이 "개인 데이터"로 표시될지 여부에 영향을 줄 수 있는 관리자 작업에 따라 달라질 수 있습니다. 예를 들어 데이터 주체가 작성하지 않은 파일을 마지막으로 수정한 경우 해당 파일이 검색 결과에 반환되지 않을 수 있습니다. 마찬가지로 관리자는 부분적으로 인덱싱된 항목이나 모든 버전의 SharePoint 문서를 포함하지 않고 데이터를 내보낼 수 있습니다. 따라서 제공된 도구는 데이터 요청에 액세스하고 내보내는 데 도움이 될 수 있습니다. 그러나 결과에는 특정 관리자 및 데이터 주체 사용 시나리오가 적용될 수 있습니다. 
  
## <a name="step-1-assign-ediscovery-permissions-to-potential-case-members"></a>1단계: 잠재적인 사례 구성원에게 eDiscovery 권한 할당

기본적으로 전역 관리자는 보안 및 준수 센터에서 DSR 사례 도구에 & 있습니다. 기본적으로 데이터 개인 정보 관리자, 인사 관리자 또는 DSR 조사에 관련된 다른 사용자와 같은 다른 사용자는 DSR 사례 도구에 액세스할 수 없습니다. 이 도구에 액세스하려면 적절한 사용 권한을 할당해야 합니다. 가장 쉬운 방법은 보안 및 준수 **Permissions** 센터의 사용 권한 페이지로 이동하여 & 관리자 역할 그룹에 사용자를 추가하는 것입니다. 또한 이러한 사용 권한을 2단계에서 만든 DSR 사례의 구성원으로 추가할 수 있도록 할당해야 합니다. 
  
단계별 지침은 Office 365 보안 및 준수 센터에서 [eDiscovery 권한 할당을 & 참조하세요.](https://docs.microsoft.com/microsoft-365/compliance/assign-ediscovery-permissions)
  
> [!NOTE]
> 기본적으로 전역 관리자(또는 Security & 준수 센터의 조직 관리 역할 그룹의 다른 구성원은 콘텐츠 검색 결과를 내보내는 데 필요한 권한이 없습니다(이 문서의 4단계 참조). 이 문제를 해결하기 위해 관리자는 자신을 eDiscovery 관리자 역할 그룹의 구성원으로 추가할 수 있습니다. 
  
## <a name="step-2-create-a-dsr-case-and-add-members"></a>2단계: DSR 사례 만들기 및 구성원 추가

다음 단계는 DSR 사례를 만드는 것입니다. 사례를 만들 때 기본 제공 검색을 시작하거나 검색을 시작하지 않고 사례를 만들 수 있습니다. 다음 절차에서는 검색을 시작하지 않고 사례를 만든 다음 사례에 구성원을 추가하는 방법을 보여 주면 됩니다.
  
1. Go to [https://protection.office.com](https://protection.office.com) and sign in using your work or school account. 
    
2. 보안 & 준수 센터에서 데이터 **Data privacy** 개인 정보 데이터 주체 요청을 클릭한 다음 아이콘 새 \> **Data subject requests** DSR 사례 ![ ](../media/ITPro-EAC-AddIcon.gif) **추가를 클릭합니다.**
    
3. 새 **DSR** 사례 플라이아웃 페이지에서 사례에 이름을 지정하고 선택적 설명을 입력한 후 다음을 **클릭합니다.** 사례 이름은 조직에서 고유해야 합니다.
    
    > [!TIP]
    > 조사할 DSR 요청을 제출한 사람의 이름 및/또는 새 사례에 대한 설명을 추가하는 것이 있습니다. 이 사례의 구성원(및 eDiscovery 관리자)만 데이터 주체 요청 페이지의 사례 목록에서 사례를 볼 **수** 있습니다. 
  
4. 요청 **세부** 정보 페이지의 데이터 주체(이 요청을 제출한 **사람)에서** 데이터를 찾아 내보낼 사람을 선택하고 다음을 **클릭합니다.**
    
5. 사례 설정 확인 **페이지에서** 사례 이름 및 설명을 변경하고 다른 데이터 주체를 선택할 수 있습니다. 그렇지 않으면 저장을 **클릭합니다.**
    
    새 DSR 사례가 만들어졌다는 확인 페이지가 표시됩니다.
    
    ![검색을 시작하거나 새 DSR 사례 페이지를 닫습니다.](../media/b5e62c2c-cafe-4a8d-a38c-789ed9f9ccbd.png)
  
    이제 다음 두 가지 중 하나를 할 수 있습니다.
    
    a. 검색 결과 **표시를 클릭하면 검색이** 시작됩니다. 이 옵션은 기본 선택 합니다. 이 옵션을 선택하면 실행되는 기본 제공 검색과 반환되는 결과가 3단계에서 설명됩니다.
    
    b. Finish를 **Finish** 클릭하면 기본 제공 검색을 시작하지 않고 새 DSR 사례가 닫히게됩니다. 이 옵션을 선택하면 데이터 주체 요청 페이지에 새 DSR **사례가** 표시됩니다.
    
6. **완료를** 클릭하여 새 DSR 사례로 이동하고 구성원을 추가할 수 있습니다. 
    
7. 데이터 **주체 요청 페이지에서** 만든 DSR 사례의 이름을 클릭합니다. 
    
8. 이 사례 **관리 플라이아웃** 페이지의 구성원 관리에서 **추가를** **클릭합니다.** 
    
    사용자 **아래에** 적절한 eDiscovery 권한이 할당된 사용자 목록이 표시됩니다. 1단계에서 eDiscovery 권한을 할당한 사람이 이 목록에 표시됩니다. 
    
9. DSR 사례의 구성원으로 추가할 대상을 **Add** 선택하고 추가를 클릭한 다음 변경 내용을 저장합니다.
    
    역할 그룹 관리에서 추가를 클릭하여 역할 그룹을 DSR 사례의 구성원으로 **추가할** **수 있습니다.** 
    
## <a name="step-3-run-the-search-query"></a>3단계: 검색 쿼리 실행

DSR 사례를 만들고 구성원을 추가한 후 다음 단계는 사례와 연결된 기본 제공 검색을 실행하는 것입니다. 이 기본 검색 쿼리는 다음을 실행합니다.
  
- 조직의 모든 사서함에서 데이터 주체가 보내거나 받은 모든 전자 메일 항목을 검색합니다. 이는 전자 메일 메시지의 모든 사용자 필드에서 데이터 주체를 검색하는  *Participants*  전자 메일 속성을 사용하여 수행됩니다. 이 속성은 데이터 주체가 **From,** **To,** **CC** 및 **BCC** 필드에 있는 항목을 반환합니다. Exchange Online의 공용 폴더는 데이터 주체가 보내거나 받은 메시지도 검색됩니다. 
    
- 조직의 모든 사이트에서 데이터 주체가 만들거나 업로드한 문서 및 항목을 검색합니다. 이 작업을 수행하기 위해 다음 사이트 속성을 사용할 수 있습니다.
    
  - *Author* 속성은 Office 문서의 만든 이 필드에 데이터 주체가 나열되는 항목을 반환합니다. 이 값은 다른 사용자가 문서를 복사하여 업로드하는 경우에도 지속됩니다. 
    
  - *CreatedBy* 속성은 데이터 주체가 만들거나 업로드한 항목을 반환합니다. 
    
다음은 DSR 사례를 만들 때 자동으로 생성되는 기본 제공 검색에 대한 키워드 쿼리의 모양입니다.
  
```powershell
participants:"<email address>" OR author:"<display name>" OR createdby:"<display name>"
```

예를 들어 데이터 주체의 이름이 Ina Leonte인 경우 키워드 쿼리는 다음과 같습니다.
  
```powershell
participants:"ina@contoso.com" OR author:"Ina Leonte" OR createdby:"Ina Leonte"
```

 **DSR 사례에 대한 기본 제공 검색을 실행합니다.**
  
1. 보안 & 준수 센터에서 데이터 **Data privacy** 개인 정보 보호 데이터 주체 요청을 클릭한 다음 2단계에서 만든 DSR 사례 옆에 있는 열기를 \> **Data subject requests** 클릭합니다. **Open** 
    
    페이지 **위쪽의** 검색 탭을 클릭한 다음 DSR 사례를 만들 때 만든 기본 제공 검색 옆의 확인란을 클릭합니다. 검색 이름은 DSR 사례와 동일합니다. 
    
2. 검색 플라이아웃 페이지에서 쿼리 **열기를 클릭합니다.**
    
    쿼리를 열면 검색이 시작된 후 몇 분 후에 완료됩니다. 
    
3. 검색이 완료되면 결과 미리 **보기를** 클릭하여 검색 결과를 미리 볼 수 있습니다. 자세한 내용은 검색 결과 [미리 보기를 참조하세요.](https://docs.microsoft.com/microsoft-365/compliance/content-search#preview-search-results)
    
    > [!TIP]
    > 검색 쿼리 통계를 확인하여 검색에서 반환되는 사서함 및 사이트 항목의 수와 검색 쿼리와 일치하는 항목이 포함된 최상위 콘텐츠 위치를 볼 수도 있습니다. 자세한 내용은 검색에 대한 정보 [및 통계 보기를 참조하세요.](https://docs.microsoft.com/microsoft-365/compliance/content-search#view-information-and-statistics-about-a-search) 
  
기본 제공 검색 쿼리를 편집하고 검색되는 콘텐츠 위치를 변경한 다음 검색을 다시 실행하면 됩니다. 자세한 [내용은 5단계를](#optional-step-5-revise-the-built-in-search-query) 참조하세요. 
  
## <a name="step-4-export-the-data"></a>4단계: 데이터 내보내기

기본 제공 검색을 실행한 후 검색 결과를 내보낼 수 있습니다. 또는 데이터를 내보내기 전에 쿼리를 변경하여 검색 결과 수를 줄일 수 있습니다. 검색 결과 범위를 좁히는 자세한 내용은 5단계를 참조하세요.
  
검색 결과를 내보낼 때 사서함 항목을 PST 파일 또는 개별 메시지로 다운로드할 수 있습니다. SharePoint 및 OneDrive 계정에서 콘텐츠를 내보내면 기본 Office 문서 및 기타 문서의 복사본이 내보내됩니다. 내보낼 모든 항목에 대한 정보가 포함된 결과 파일이 검색 결과에 포함됩니다. 내보내기에 대한 자세한 내용은 콘텐츠 검색 결과 [내보내기를 참조하세요.](https://docs.microsoft.com/microsoft-365/compliance/export-search-results)
  
> [!NOTE]
> 기본적으로 전역 관리자(또는 Security & 준수 센터의 조직 관리 역할 그룹의 다른 구성원)에게는 콘텐츠 검색 결과를 내보내는 데 필요한 권한이 없습니다. 이 문제를 해결하기 위해 관리자는 자신을 eDiscovery 관리자 역할 그룹의 구성원으로 추가할 수 있습니다. 
  
데이터를 내보내는 데 사용하는 컴퓨터는 다음 시스템 요구 사항을 충족해야 합니다.
  
- 32비트 또는 64비트 버전의 Windows 7 이상 버전
    
- Microsoft .NET Framework 4.7
    
- 지원되는 브라우저:
    
  - Microsoft Edge
    
    또는
    
  - Microsoft Internet Explorer 10 이상 버전
    
    > [!NOTE]
    > Microsoft는 타사 응용 프로그램용 타사 확장 또는 ClickOnce 않습니다. 타사 확장 또는 추가 기능이 있는 지원되지 않는 브라우저를 사용하여 데이터를 내보내는 것은 지원되지 않습니다. 
  
 **DSR 사례에서 기본 제공 검색에서 데이터를 내보내는 경우:**
  
1. 보안 & 준수 센터에서 데이터 **Data privacy** 개인 정보 보호 데이터 주체 요청을 클릭한 다음 데이터를 내보낼 DSR 사례 옆에 있는 \> **Data subject requests** 열기를 **Open** 클릭합니다. 
    
2. 페이지 **위쪽의** 검색 탭을 클릭한 다음 DSR 사례를 만들 때 만든 기본 제공 검색 옆의 확인란을 클릭합니다. 또는 다른 검색을 클릭하여 해당 검색에서 데이터를 내보낼 수 있습니다. 
    
3. 검색 플라이아웃 페이지에서 검색 결과 내보내기 아이콘 더하기를 클릭한 다음 드롭다운 목록에서 결과 ![ ](../media/47205c65-babd-4b3a-bd7b-98dfd92883ba.png) **More** 내보내기를 **Export results** 선택합니다. 
    
4. 결과 **내보내기 페이지에서** DSR 내보내기 요청에 대한 다음 권장 옵션을 선택합니다. 
    
    ![내보내기 설정 구성](../media/25416b79-57da-46a1-ae07-e640602a8fa4.png)
  
    a. 출력 **옵션에서** 첫 번째 **All items, excluding ones that have ones that have an unrecognized format, are encrypted, or weren't indexed for other reasons** 옵션(인식할 수 없는 형식이 있는 항목, 암호화되거나 다른 이유로 인덱싱되지 않은 항목 제외)을 선택하여 인덱싱된 항목만 내보낼 수 있습니다. 기본 제공 검색에서 부분적으로 인덱싱된 항목을 내보내지 않는 이유는 다른 사용자의 부분적으로 인덱싱된 항목도 내보내기 때문에입니다. 데이터 주체에 대해 부분적으로 인덱싱된 항목만 내보내기 위해 별도의 검색을 만드는 것이 좋습니다. 자세한 내용은 "DSR 사례 도구 사용에 대한 자세한 정보" 섹션에서 부분적으로 인덱싱된 항목 내보내기를 참조하세요. [Exporting partially indexed items](#exporting-partially-indexed-items)
    
    b. Exchange **콘텐츠 내보내기 아래에서** 세 번째 옵션인 단일 폴더에 있는 모든 메시지를 포함하는 **PST 파일 하나를 선택합니다.** 결과 중 일부는 다른 사용자의 사서함에서 시작된 항목에 대한 것일 수 있기 때문에 이 옵션은 실제 사서함을 나타내지 않고 단일 폴더에 항목을 나열하기만 하여 다음 항목에 권장되는 결과의 중복을 제거하면 가장 좋은 옵션입니다. 또한 이 옵션을 사용하면 데이터 주체가 각 항목의 원래 사서함 폴더 구조를 탐색하지 않고도 항목을 시간 순서(항목이 보낸 날짜별로 정렬)로 검토할 수 있습니다.
    
    c. 중복 **전자 메일 메시지를** 제외하려면 중복 제거 사용 옵션을 선택합니다. 기본 제공 검색은 조직의 모든 사서함을 검색하기 때문에 이 옵션을 사용하는 것이 좋습니다. 따라서 검색된 사서함에서 동일한 메시지의 여러 복사본이 발견되는 경우 이 옵션은 메시지 복사본을 하나만 내보낼 수 있도록 합니다. 이 옵션은 단일 폴더에 있는 하나의 PST 파일로 메시지를 함께 내보내며, DSR 내보내기 요청에 가장 적합한 사용자 환경을 제공합니다. 이 Results.csv 내보내기 보고서에는 중복된 메시지가 발견된 모든 위치가 나열됩니다.
    
    원하는 경우 SharePoint 문서에 대한 버전 포함 옵션을 선택하여 모든 버전의 **SharePoint** 및 OneDrive 문서를 내보낼 수 있습니다. 이렇게 하려면 문서 라이브러리에 대해 버전 관리가 켜져야 합니다. 이 옵션은 모든 관련 데이터를 내보내는 데 도움이 됩니다.
    
5. 내보내기 설정을 선택한 후 내보내기 를 **클릭합니다.**
    
    검색 결과는 다운로드할 수 있도록 준비됩니다. 즉, Microsoft 클라우드의 조직의 Azure Storage 영역에 업로드됩니다. 다음 단계에서는 이 데이터를 로컬 컴퓨터에 다운로드하는 방법을 보여 제공합니다.
    
6. 내보내기 **탭을** 클릭하여 만든 내보내기 작업을 표시합니다. 내보내기 작업의 이름은 검색 이름 **_Export** 끝에 추가된 _Export 해당 검색과 동일합니다. 
    
7. 방금 만든 내보내기 작업을 클릭하여 내보내기 플라이아웃 페이지를 표시합니다. 이 페이지에는 내보낼 총 항목 수 및 Azure 저장소 영역으로 전송된 항목의 백분율과 같은 검색 정보가 표시됩니다. 새로 **고침을** 클릭하여 업로드 상태 정보를 업데이트합니다. 
    
8. **내보내기 키** 에서 **클립보드로 복사** 를 클릭합니다. 11단계에서 이 키를 사용하여 검색 결과를 다운로드합니다.
    
9. 내보내기 플라이아웃 페이지의 맨 위에 있는 검색 결과 내보내기 아이콘 ![ ](../media/47205c65-babd-4b3a-bd7b-98dfd92883ba.png) **Download results** 다운로드를 클릭합니다. 
    
10. 페이지 아래쪽의 팝업 창에서 열기를 **Open** 클릭하여 **eDiscovery 내보내기 도구를 니다.** 검색 결과를 처음 다운로드하면 **eDiscovery** 내보내기 도구가 설치됩니다. 
    
11. **eDiscovery 내보내기** 도구에서 8단계에서 복사한 내보내기 키를 적절한 상자에 붙여 넣습니다.
    
12. **찾아보기** 를 클릭하여 검색 결과 파일을 다운로드하려는 위치를 지정합니다. 
    
    > [!NOTE]
    > 읽기 및 쓰기가 많은 디스크 작업으로 인해 검색 결과를 로컬 디스크 드라이브에 다운로드해야 합니다. 매핑된 네트워크 드라이브 또는 다른 네트워크 위치에 다운로드하지 않습니다. 
  
13. **시작** 을 클릭하여 컴퓨터에 검색 결과를 다운로드합니다. 
    
    **eDiscovery 내보내기 도구** 는 다운로드할 남은 항목의 예상 개수(크기)를 포함하여 내보내기 프로세스에 대한 상태 정보를 표시합니다. 내보내기 프로세스가 완료되면 다운로드된 위치에 있는 파일에 액세스할 수 있습니다. 콘텐츠 검색 결과를 다운로드할 때 포함된 보고서에 대한 자세한 내용은 "콘텐츠 검색 결과 내보내기"의 [추가](https://docs.microsoft.com/microsoft-365/compliance/export-search-results#more-information) 정보 섹션을 참조하세요. 
    
데이터를 내보낼 때 검색 결과 및 내보내기 보고서는 DSR 사례와 이름이 같은 폴더에 있습니다. 사서함 항목을 포함하는 PST 파일은 Exchange라는 하위 폴더에 **있습니다.** 사이트의 문서 및 기타 항목은 **SharePoint라는 하위폴더에 있습니다.** 
  
## <a name="optional-step-5-revise-the-built-in-search-query"></a>(선택 사항) 5단계: 기본 제공 검색 쿼리 변경

기본 제공 검색을 실행한 후 검색 결과를 더 적게 반환하기 위해 범위를 좁히기 위해 검색 범위를 좁힐 수 있습니다. 이 작업을 수행하기 위해 쿼리에 조건을 추가하면 됩니다. 조건은 AND 연산자에 의해 키워드 쿼리에 논리적으로 **연결됩니다.** 즉, 검색 결과에 반환되는 항목은 키워드 쿼리와 추가한 조건을 모두 충족해야 합니다. 조건이 결과 범위를 좁히는 데 도움이 되는 방식입니다. 검색 쿼리에 둘 이상의 고유한 조건(서로 다른 속성을 지정하는 조건)을 추가하는 경우 해당 조건은 AND 연산자에 의해 논리적으로 **연결됩니다.** 즉, 키워드 쿼리 외에 모든 조건을 충족하는 항목만 반환됩니다. 단일 조건에 여러 값을 추가(콤보 또는 세미 콜론으로 구분)하는 경우 이러한 값은 OR 연산자에 **의해 연결됩니다.** 즉, 조건의 속성에 대해 지정된 값을 하나라도 포함하는 항목이 반환됩니다. 
  
다음은 DSR 사례의 기본 제공 검색 쿼리에 추가할 수 있는 조건의 몇 가지 예입니다. 검색 쿼리에 사용된 실제 속성의 이름은 괄호로 표시됩니다.
  
- **파일 형식( `filetype` )** – 문서 또는 파일의 확장명을 지정합니다. 이 조건을 사용하여 Word, Excel 및 OneNote와 같은 특정 Office 응용 프로그램에서 만든 문서 및 파일을 검색할 수 있습니다. 
    
- **메시지 유형( `kind` )** – 검색할 전자 메일 항목의 유형을 지정합니다. 예를 들어 구문을 사용하여 Microsoft Teams에서 전자 메일 메시지와 비즈니스용 Skype 대화 또는 일대일 채팅만 반환할  `kind:email OR kind:im` 수 있습니다. 
    
- **준수 태그( `compliancetag` )** - 전자 메일 메시지 또는 문서에 할당된 레이블을 지정합니다. 이 조건은 특정 레이블로 분류된 항목을 반환합니다. 레이블은 데이터 거버넌스를 위해 전자 메일 및 문서를 분류하고 레이블로 정의된 분류에 따라 보존 규칙을 적용하는 데 사용됩니다. 이는 조직에서 레이블을 사용하여 데이터 개인 정보와 관련된 콘텐츠 또는 개인 데이터 또는 중요한 정보를 포함하는 콘텐츠를 분류할 수 있기 때문에 DSR 조사에 유용한 조건입니다. 이 조건의 값으로는 전체 레이블 이름 또는 레이블 이름의 첫 번째 부분을 와일드카드와 함께 사용하십시오. 자세한 내용은 [Office 365의](https://docs.microsoft.com/microsoft-365/compliance/retention)보존 정책 및 보존 레이블에 대한 자세한 내용을 참조하세요.
    
DSR 사례 도구에서 사용할 수 있는 모든 조건에 [Search conditions](https://docs.microsoft.com/microsoft-365/compliance/keyword-queries-and-search-conditions#search-conditions) 대한 목록 및 설명은 "콘텐츠 검색에 대한 키워드 쿼리 및 검색 조건" 문서의 검색 조건을 참조하세요. 
  
### <a name="changing-the-content-locations-that-are-searched"></a>검색되는 콘텐츠 위치 변경

DSR 사례에 대한 기본 제공 검색을 변경하는 것 외에도 검색되는 콘텐츠 위치를 변경할 수도 있습니다. 앞서 설명한 것 처럼 기본 제공 검색은 조직의 모든 사서함 및 사이트와 모든 Exchange Online 공용 폴더를 검색합니다. 예를 들어 검색 범위를 좁혀 데이터 주체의 사서함과 OneDrive 계정 및 선택한 SharePoint 사이트만 검색할 수 있습니다. 특정 사이트를 검색하도록 선택한 경우 검색할 각 사이트를 추가해야 합니다.
  
검색할 콘텐츠 위치를 수정하려면
  
1. 콘텐츠 위치를 변경할 기본 제공 검색을 열 수 있습니다.
    
2. 검색 쿼리의 **위치** 아래에서 특정 위치 **옵션** 옆에 있는 **수정을** 클릭합니다. 
    
    ![수정을 클릭하여 기본 제공 검색 쿼리의 콘텐츠 위치 변경](../media/d66f7ba7-b71f-4ff5-a030-460ff02e3123.png)
  
    위치 **수정 플라이아웃** 페이지가 표시됩니다. 다음은 기본 제공 검색의 콘텐츠 위치에 대한 설명과 검색되는 위치를 수정하는 데 대한 몇 가지 정보입니다. 
    
    ![위치 플라이아웃 페이지 수정](../media/56c033f6-6735-46ba-abb2-a263a2b79836.png)
  
    a. 플라이아웃 페이지 **Select all** 맨 위에 있는 모든 사서함 선택 섹션의 토글이 선택되어 모든 사서함이 검색됩니다. 검색 범위를 좁히려면 토글을 클릭하여 선택을 제거한 다음 **사용자,** 그룹 또는 팀 선택을 클릭하고 검색할 특정 사서함을 선택하세요.
    
    b. 플라이아웃 페이지 **Select all** 중간의 모든 사이트 선택 섹션에서 토글이 선택되어 모든 사이트가 검색됩니다. 선택한 사이트로 검색 범위를 좁히려면 토글을 선택하지 않은 다음 사이트 **선택을 클릭합니다.** 데이터 주체의 OneDrive 계정을 포함하여 검색할 각 특정 사이트를 추가해야 합니다.
    
    c. Exchange 공용 폴더 섹션의 토글이 선택되어 모든 Exchange 공용 폴더가 검색됩니다. 모든 Exchange 공용 폴더만 검색할 수도 있습니다. 검색할 특정 옵션을 선택할 수 없습니다.
    
3. 기본 제공 검색에서 콘텐츠 위치를 수정하는 **경우 &amp;** 저장 실행을 클릭하여 검색을 다시 시작합니다. 

> [!NOTE]
> 모든 사서함 위치 또는 특정 사서함만 검색하면 검색 결과를 내보낼 때 사용자 사서함에 저장된 다른 Office 365 응용 프로그램의 데이터가 포함됩니다. 이 데이터는 예상 검색 결과에 포함되지 않으며 미리 보기에서 확인할 수 없습니다. 그러나 검색 결과를 내보내고 다운로드할 때 포함됩니다. 사용자의 사서함에 데이터를 저장하는 응용 프로그램에 대한 자세한 내용은 Exchange Online 사서함에 [저장된 콘텐츠를 참조하십시오.](https://docs.microsoft.com/microsoft-365/compliance/what-is-stored-in-exo-mailbox)

### <a name="exporting-data-from-the-office-roaming-service"></a>Office 로밍 서비스에서 데이터 내보내기

DSR 사례 도구를 사용하여 Office 로밍 서비스에서 생성된 사용 현황 데이터를 검색하고 내보낼 수 있습니다. 로밍은 Office 테마, 사용자 지정 사전, 언어 설정, 개발자 모드 및 자동 수정과 같은 Office 관련 설정을 저장하는 서비스입니다. 
    
Office 로밍 서비스의 데이터는 Exchange Online 사서함의 비 IPM(비인간 메시지) 하위 위치에 있는 숨겨진 폴더에 데이터 주체의 사서함에 저장됩니다. 즉, Outlook 또는 다른 메일 클라이언트를 사용하여 사서함에 액세스할 때 사용자의 보기에서 데이터가 숨겨집니다. 숨겨진 폴더에 대한 자세한 내용은 [MAPI 숨겨진 폴더를 참조하세요.](https://go.microsoft.com/fwlink/?linkid=872758)
  
데이터 주체의 사서함에서 Office 로밍 서비스 사용 현황 데이터를 반환하는 별도의 콘텐츠 검색을 만들어 DSR 사례와 연결합니다. 이 데이터는 검색 통계에 포함되지 않습니다. 미리 보기에 사용할 수 없습니다. 그러나 내보낼 수 있으며 DSR 내보내기 요청에 대한 응답으로 데이터 주체에게 제공하면 됩니다.
  
Office 로밍 서비스에서 데이터를 내보낼 때 데이터는 **ApplicationDataRoot** 폴더에 있는 별도의 폴더에 저장됩니다. 이 폴더는 데이터 주체의 전자 메일 주소가 있는 폴더 아래에 있습니다. 이 데이터는 전자 메일 메시지에 첨부된 XML 또는 TXT 파일과 유사한 사람이 읽을 수 있는 텍스트 파일인 JSON 파일로 내보낼 수 있습니다. 현재 이 폴더의 이름은 GUID(Globally Unique **Identifier):1caee58f-eb14-4a6b-9339-1fe2ddf6692b입니다.** DSR 사례 도구의 향후 버전에서는 GUID가 실제 응용 프로그램의 이름으로 대체됩니다. 

   
 **Office 로밍 서비스 데이터를 검색하고 내보내는 경우:**
  
1. 보안 & 준수 센터에서 데이터 **Data privacy** 개인 정보 보호 데이터 주체 요청을 클릭한 다음 사용 현황 데이터를 내보낼 데이터 주체에 대한 DSR 사례 옆에 있는 열기를 \> **Data subject requests** 클릭합니다. **Open** 
    
2. 페이지 맨 **위에** 있는 검색 탭을 클릭한 다음 아이콘 안내 검색 ![ ](../media/ITPro-EAC-AddIcon.gif) **추가를 클릭합니다.**
    
3. 검색 **이름** 페이지에서 **취소를** 클릭합니다. 
    
4. 검색 **쿼리의** **유형** 조건에서 Office 로밍 서비스 옆에 있는 **확인란을 선택합니다.** 
    
    ![Office 로밍 서비스 확인란을 선택하여 사용 현황 데이터 내보내기](../media/O365_DSRCase_SDSDataExport1.png)
  
    형식 **조건(전자** 메일 메시지 클래스)은 검색 쿼리에서 유일한 항목입니다. 키워드 **상자를** 삭제하거나 비워 두면 됩니다. 
    
5. 위치 **아래에서** 특정 **위치가** 선택되어 있는지 확인한 다음 수정을 **클릭합니다.**
    
6. 위치 수정 플라이아웃 페이지(사서함 섹션)의 위쪽에서 사용자, 그룹 또는 팀 **선택을 클릭합니다.** **Modify locations**
    
7. 위치 **편집 페이지에서** **사용자,** 그룹 또는 팀 선택을 클릭하고 데이터 주체의 사서함을 선택한 다음 선택을 저장합니다. 
    
8. 실행 & 저장을 **클릭한** 다음 검색 이름을 지정하고 저장합니다.
    
    검색이 시작됩니다.
    
 **Office 로밍 서비스 데이터를 내보내는 경우:**
  
1. 이전 단계에서 만든 검색이 완료되면 페이지 맨 **Search** 위에 있는 검색 탭을 클릭한 다음 검색 옆의 확인란을 클릭합니다. 검색을 표시하려면 새로 ![ ](../media/165fb3ad-38a8-4dd9-9e76-296aefd96334.png) **고침을** 클릭해야 할 수 있습니다. 
    
2. 검색 플라이아웃 페이지에서 검색 결과 내보내기 아이콘 더하기를 클릭한 다음 드롭다운 목록에서 결과 ![ ](../media/47205c65-babd-4b3a-bd7b-98dfd92883ba.png) **More** 내보내기를 **Export results** 선택합니다. 
    
3. 결과 **내보내기 페이지에서** 사용 현황 데이터를 내보내는 권장 옵션을 선택합니다. 
    
    ![Office 로밍 서비스 사용 현황 데이터를 내보낼 때 옵션 내보내기](../media/470a7d1e-eeae-4b42-95aa-15cb82ce2f68.png)
  
    a. 출력 **옵션에서** 첫 번째 **All items, excluding ones that have ones that have an unrecognized format, are encrypted, or weren't indexed for other reasons** 옵션(인식할 수 없는 형식이 있는 항목, 암호화되거나 다른 이유로 인덱싱되지 않은 항목 제외)을 선택하여 인덱싱된 항목만 내보낼 수 있습니다.
    
    b. Exchange **콘텐츠 내보내기 아래에서** 두 번째 옵션인 모든 메시지를 포함하는 PST 파일 **하나를 선택합니다.**
    
    c. 나머지 내보내기 옵션을 선택하지 않은 나머지로 떠 습니다.
    
4. 내보내기 설정을 선택한 후 내보내기 를 **클릭합니다.**
    
    검색 결과는 다운로드할 수 있도록 준비됩니다. 즉, Microsoft 클라우드의 조직의 Azure 저장소 영역에 업로드됩니다. 다음 단계에서는 이 데이터를 로컬 컴퓨터에 다운로드하는 방법을 보여 제공합니다.
    
5. 내보내기 **탭을** 클릭하여 만든 내보내기 작업을 표시합니다. 내보내기 작업의 이름은 검색 이름 끝에 추가된 _Export 검색과 동일합니다. **_Export** 
    
6. 방금 만든 내보내기 작업을 클릭하여 내보내기 플라이아웃 페이지를 표시합니다. 
    
7. **내보내기 키** 에서 **클립보드로 복사** 를 클릭합니다. 10단계에서 이 키를 사용하여 검색 결과를 다운로드합니다.
    
8. 내보내기 플라이아웃 페이지의 맨 위에 있는 검색 결과 내보내기 아이콘 ![ ](../media/47205c65-babd-4b3a-bd7b-98dfd92883ba.png) **Download results** 다운로드를 클릭합니다. 
    
9. 페이지 아래쪽의 팝업 창에서 열기를 **Open** 클릭하여 **eDiscovery 내보내기 도구를 니다.** 검색 결과를 처음 다운로드하면 **eDiscovery** 내보내기 도구가 설치됩니다. 
    
10. **eDiscovery 내보내기 도구** 에서 7단계에서 복사한 내보내기 키를 해당 상자에 붙여 넣습니다.
    
11. **찾아보기** 를 클릭하여 검색 결과 파일을 다운로드하려는 위치를 지정합니다. 
    
    > [!NOTE]
    > 읽기 및 쓰기가 많은 디스크 작업으로 인해 검색 결과를 로컬 디스크 드라이브에 다운로드해야 합니다. 매핑된 네트워크 드라이브 또는 다른 네트워크 위치에 다운로드하지 않습니다. 
  
12. **시작** 을 클릭하여 컴퓨터에 검색 결과를 다운로드합니다. 
    
    **eDiscovery 내보내기 도구** 는 다운로드할 남은 항목의 예상 개수(크기)를 포함하여 내보내기 프로세스에 대한 상태 정보를 표시합니다. 내보내기 프로세스가 완료되면 Outlook에서 Exchange PST 파일을 연 다음 **ApplicationDataRoot** 폴더로 이동하여 로밍 서비스의 하위 폴더에 액세스할 수 있습니다. 
    
    앞서 설명한 것 처럼 사용 현황 데이터가 포함된 JSON 파일은 메시지에 첨부됩니다. JSON 파일을 보려면 메시지를 클릭한 다음 첨부된 JSON 파일을 열 수 있습니다. 
  
### <a name="exporting-partially-indexed-items"></a>부분적으로 인덱싱된 항목 내보내기

DSR 사례를 만들 때 만들어진 기본 제공 검색에서 부분적으로 인덱싱된 항목(인덱싱되지 않은 항목이라고도 합니다)을 내보내지 않는 것이 좋습니다. 이는 검색 결과에 데이터 주체에 대한 부분적으로 인덱싱된 항목만 포함되는 것이 아니라 조직의 다른 사용자에 대해 부분적으로 인덱싱된 항목이 포함될 가능성이 높기 때문에입니다. 대신 데이터 주체와 관련된 부분적으로 인덱싱된 항목만 내보내기 위한 DSR 사례와 연결된 별도의 콘텐츠 검색을 만드는 것이 좋습니다. 
  
다음은 부분적으로 인덱싱된 항목을 내보내는 고급 프로세스입니다. 내보내기 후 항목을 검토하여 항목이 DSR 액세스 또는 내보내기 요청에 응답하는지 확인할 수 있습니다.
  
1. DSR 사례를 열고 검색 페이지에서 **검색을** 만드십시오. 
    
2. 검색할 검색 쿼리 및 콘텐츠 위치를 구성하는 데는 다음 조건을 사용합니다.
    
    - 빈 키워드 쿼리를 사용합니다. 그러면 검색되는 콘텐츠 위치의 모든 항목이 반환됩니다.
    
    - 데이터 주체의 Exchange Online 사서함과 해당 OneDrive 계정만 검색합니다.
    
3. 검색을 실행하고 완료한 후 검색 결과를 내보내고 다운로드할 수 [있습니다(4단계에](#step-4-export-the-data)설명). 다음 설정을 사용하여 부분적으로 인덱싱된 항목을 내보낼 수 있습니다. 
    
    - 출력 **옵션에서** 세 번째 **Only items that have an unrecognized format, are encrypted, or weren't indexed for other reasons** 옵션(인식할 수 없는 형식이거나 암호화되거나 다른 이유로 인덱싱되지 않은 항목만)을 선택하여 부분적으로 인덱싱된 항목만 내보낼 수 있습니다.
    
    - Exchange **콘텐츠 내보내기 아래에서** 기본 설정에 따라 원하는 옵션을 선택할 수 있습니다. 
    
    - **SharePoint 문서에** 대한 버전 포함 옵션을 선택하면 버전이 부분적으로 인덱싱된 경우 문서 버전이 내보내지 않습니다. 
    
부분적으로 인덱싱된 항목에 대한 자세한 내용은 다음을 참조하세요. 
  
- [Office 365의 콘텐츠 검색에서 부분적으로 인덱싱된 항목](https://docs.microsoft.com/microsoft-365/compliance/partially-indexed-items-in-content-search)

- [부분적으로 인덱싱된 항목 내보내기](https://docs.microsoft.com/microsoft-365/compliance/export-search-results#exporting-partially-indexed-items)

### <a name="searching-and-exporting-data-from-microsoft-teams-and-microsoft-365-groups"></a>Microsoft Teams 및 Microsoft 365 그룹에서 데이터 검색 및 내보내기

Microsoft Teams의 채팅 목록에 있는 대화(팀 채팅 또는 일대일 채팅이라고도 함)는 채팅에 참여하는 사용자의 Exchange Online 사서함에 저장됩니다. 또한 일대일 채팅에서 공유하는 파일은 파일을 공유하는 사람의 OneDrive 계정에 저장됩니다. 기본 제공 검색은 조직의 모든 사서함 및 OneDrive 계정을 검색하기 때문에 채팅 세션에서 공유된 팀 채팅 및 문서(데이터 주체가 만들거나 업로드한)는 DSR 사례의 기본 제공 검색에서 반환됩니다.
  
또는 Teams 채널의 일부인 대화(채널 메시지라고도함)는 팀과 연결된 사서함에 저장됩니다. Teams와 연결된 모든 사서함이 검색되어 데이터 주체가 참여한 이러한 유형의 대화도 기본 제공 검색에서 반환됩니다. 또한 데이터 주체가 Teams 채널에서 공유하는 파일은 팀의 SharePoint 사이트에 저장됩니다. 데이터 주체가 만들거나 업로드한 파일은 Teams와 연결된 사이트가 검색에 포함되어 있기 때문에 DSR 사례의 기본 제공 검색에서 반환됩니다.
  
마찬가지로 Microsoft 365 그룹에 해당하는 사서함 및 SharePoint 사이트도 기본 제공 검색에 포함됩니다. 즉, 데이터 주체가 보내거나 받은 전자 메일 메시지와 데이터 주체가 만들거나 업로드한 파일이 반환됩니다. 
  
콘텐츠 검색을 사용하여 Microsoft Teams 및 Microsoft 365 그룹에서 항목을 검색하거나 구성원 목록을 다운로드하는 방법에 대한 자세한 내용은 [Microsoft 365의](https://docs.microsoft.com/microsoft-365/compliance/content-search#searching-microsoft-teams-and-microsoft-365-groups)콘텐츠 검색에서 "Microsoft Teams 및 Microsoft 365 그룹 검색" 섹션을 참조하세요. 
  
### <a name="searching-exchange-public-folders"></a>Exchange 공용 폴더 검색

DSR 사례의 기본 제공 검색은 데이터 주체가 메일 사용이 가능한 공용 폴더로 보낸 전자 메일 메시지 또는 다른 사람이 공용 폴더로 전송하고 데이터 주체도 복사한 메시지만 반환합니다. 데이터 주체가 공용 폴더에 게시한 메시지는 반환하지 않습니다. 데이터 주체가 공용 폴더에 게시한 항목을 검색하기 위해 별도의 콘텐츠 검색을 만들어 데이터 주체가 공용 폴더에 게시한 항목을 검색할 수 있습니다.
  
다음은 데이터 주체가 공용 폴더에 게시한 항목을 검색하는 고급 프로세스입니다. 
  
1. DSR 사례를 열고 검색 페이지에서 **검색을** 만드십시오. 
    
2. 검색할 검색 쿼리 및 콘텐츠 위치를 구성하는 데는 다음 조건을 사용합니다.
    
  - 키워드 **상자에서** 다음 검색 쿼리를 사용합니다. 
    
    ```powershell
    itemclass:ipm.post AND "<email address of the data subject>"
    ```

  - 모든 Exchange 공용 폴더 검색
    
  - 검색을 실행하고 완료한 후 검색 결과를 내보내고 다운로드할 수 [있습니다(4단계에](#step-4-export-the-data)설명). 다음 설정을 사용하여 부분적으로 인덱싱된 항목을 내보낼 수 있습니다.
