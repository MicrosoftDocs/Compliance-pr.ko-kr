## <a name="how-does-microsoft-365-encrypt-data-at-rest"></a><span data-ttu-id="0ce19-110">Microsoft 365는 미사용 데이터를 암호화하는 방법</span><span class="sxs-lookup"><span data-stu-id="0ce19-110">How does Microsoft 365 encrypt data-at-rest?</span></span>

<span data-ttu-id="0ce19-111">Microsoft 365의 모든 고객 콘텐츠는 하나 이상의 암호화 형태로 보호됩니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-111">All customer content in Microsoft 365 is protected by one or more forms of encryption.</span></span> <span data-ttu-id="0ce19-112">Microsoft 서버는 BitLocker를 사용하여 볼륨 수준에서 고객 콘텐츠가 포함된 디스크 드라이브를 암호화합니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-112">Microsoft servers use BitLocker to encrypt the disk drives containing customer content at the volume-level.</span></span> <span data-ttu-id="0ce19-113">BitLocker에서 제공하는 암호화는 고객 콘텐츠가 포함된 디스크에 무단으로 물리적으로 액세스할 수 있는 다른 프로세스 또는 제어(예: 하드웨어의 액세스 제어 또는 재생)에서 경과되는 경우 고객 콘텐츠를 보호합니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-113">The encryption provided by BitLocker protects customer content in case of lapses in other processes or controls (for example, access control or recycling of hardware) that could lead to unauthorized physical access to disks containing customer content.</span></span>

<span data-ttu-id="0ce19-114">Exchange Online, Microsoft Teams, SharePoint Online 및 비즈니스용 OneDrive에서도 응용 프로그램 계층의 서비스 암호화를 사용하여 고객 콘텐츠를 암호화합니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-114">Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business also use Service Encryption at the application layer to encrypt customer content.</span></span> <span data-ttu-id="0ce19-115">서비스 암호화는 강력한 암호화 보호를 통해 권한 보호 및 관리 기능을 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-115">Service Encryption provides rights protection and management features on top of strong encryption protection.</span></span> <span data-ttu-id="0ce19-116">또한 Windows 운영 체제와 해당 운영 체제에서 저장하거나 처리한 고객 데이터를 분리할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-116">It also allows for separation between Windows operating systems and the customer data stored or processed by those operating systems.</span></span>

## <a name="how-does-microsoft-365-encrypt-data-in-transit"></a><span data-ttu-id="0ce19-117">Microsoft 365는 전송되는 데이터를 암호화하는 방법</span><span class="sxs-lookup"><span data-stu-id="0ce19-117">How does Microsoft 365 encrypt data-in-transit?</span></span>

<span data-ttu-id="0ce19-118">Microsoft 365 제품 및 서비스는 TLS와 같은 강력한 전송 프로토콜을 사용하여 권한이 없는 사용자가 네트워크를 통해 이동하는 동안 고객 데이터를 도청하지 못하게 합니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-118">Microsoft 365 products and services use strong transport protocols, such as TLS, to prevent unauthorized parties from eavesdropping on customer data while it moves over a network.</span></span> <span data-ttu-id="0ce19-119">전송되는 데이터의 예로는 배달 중인 메일 메시지, 온라인 모임에서 진행되는 대화 또는 데이터 센터 간에 복제되는 파일이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-119">Examples of data in transit include mail messages that are in the process of being delivered, conversations taking place in an online meeting, or files being replicated between datacenters.</span></span>

<span data-ttu-id="0ce19-120">Microsoft 365에서 데이터는 사용자의 장치가 Microsoft 서버와 통신하거나 Microsoft 서버가 다른 서버와 통신할 때마다 '전송 중'으로 간주됩니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-120">In Microsoft 365, data is considered 'in transit' whenever a user's device is communicating with a Microsoft server or a Microsoft server is communicating with another server.</span></span> <span data-ttu-id="0ce19-121">Exchange Online, SharePoint Online, Microsoft Teams 및 Office Online은 모두 TLS를 사용하여 전송되는 동안 데이터를 기밀로 유지하도록 합니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-121">Exchange Online, SharePoint Online, Microsoft Teams, and Office Online all use TLS to ensure data remains confidential while in transit.</span></span>

## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="0ce19-122">Microsoft 365는 암호화에 사용되는 키를 어떻게 관리하나요?</span><span class="sxs-lookup"><span data-stu-id="0ce19-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="0ce19-123">강력한 암호화는 데이터를 암호화하는 데 사용되는 키만큼 안전합니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-123">Strong encryption is only as secure as the keys used to encrypt data.</span></span> <span data-ttu-id="0ce19-124">Microsoft는 자체 보안 인증서를 사용하여 전송되는 데이터에 대한 TLS 연결을 암호화합니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-124">Microsoft uses its own security certificates to encrypt TLS connections for data-in-transit.</span></span> <span data-ttu-id="0ce19-125">미사용 데이터의 경우 BitLocker로 보호되는 볼륨은 볼륨 마스터 키로 암호화되는 전체 볼륨 암호화 키로 암호화되며, 이 키는 서버의 TPM(신뢰할 수 있는 플랫폼 모듈)에 바인딩됩니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-125">For data-at-rest, BitLocker-protected volumes are encrypted with a full volume encryption key, which is encrypted with a volume master key, which in turn is bound to the Trusted Platform Module (TPM) in the server.</span></span> <span data-ttu-id="0ce19-126">BitLocker는 FIPS 호환 알고리즘을 사용하여 암호화 키가 유선으로 저장되거나 유선으로 전송되지 않도록 합니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-126">BitLocker uses FIPS-compliant algorithms to ensure that encryption keys are never stored or sent over the wire in the clear.</span></span>

<span data-ttu-id="0ce19-130">자체 루트 암호화 키를 제어해야 하는 요구 사항이 있는 고객은 고객 키로 서비스 암호화를 활용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="0ce19-131">고객은 고객 키를 사용하여 자체 암호화 키를 생성할 수 있으며, 이러한 키는 HSM(On-프레미스 하드웨어 서비스 모듈) 또는 AKV(Azure Key Vault)를 사용하여 생성할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="0ce19-132">고객 루트 키는 AKV에 저장되며 고객 사서함 데이터 또는 파일을 암호화하는 키사인 중 하나의 루트로 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="0ce19-133">고객 루트 키는 데이터 암호화를 위한 Microsoft 365 서비스 코드에서 간접적으로만 액세스할 수 있으며 Microsoft 직원은 직접 액세스할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="0ce19-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
